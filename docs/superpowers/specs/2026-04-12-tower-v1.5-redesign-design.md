**Date:** 2026-04-12

## 1. Overview

Three coordinated improvements to the Tower mobile app:

1. **Auth fix** — clear local WatermelonDB when a different user signs in (race condition in current sign-out flow)
2. **Home screen redesign** — replace Sankey chart with a swipeable tile carousel: Wellness Score tile (compact) + Total Budget tile + per-category budget tiles, week/month configurable
3. **Financial Wellness Score** — gamified score (0–100) computed from how closely actual spend matches the user's income allocation plan; drives the score tile sparkline
4. **Plan tab restructure** — Report / Planning toggle; Report = Sankey; Planning = Income Buckets (% allocations) + Savings Goals, both as expandable rows
5. **Modern floating nav bar** — pill-shaped floating tab bar with icons + labels

Sankey chart moves off the Home tab entirely and lives in Plan → Report.

## 3. Auth Fix — Deterministic DB Clear

### Problem
`AuthContext.signOut()` calls `setSession(null)` then `database.unsafeResetDatabase()`. React does not process the state update synchronously — the tab navigator (and its WatermelonDB observers) may still be mounted when the reset runs. If `unsafeResetDatabase()` throws due to active observers, the `.catch(() => {})` silently swallows it, leaving stale data in the DB. A different user who signs in on the same device will see the previous user's linked accounts.

### Fix — Reset on Sign-in, Not Sign-out
Store the last authenticated user ID in `AsyncStorage`. On every sign-in (via `onAuthStateChange` with `SIGNED_IN` event), compare the incoming user ID against the stored one. If they differ, call `database.unsafeResetDatabase()` **before** the app mounts the authenticated screens (no observers exist yet).

```
onAuthStateChange(SIGNED_IN, session):
  storedId = AsyncStorage.getItem('tower_last_user_id')
  if storedId && storedId !== session.user.id:
    await database.unsafeResetDatabase()
  AsyncStorage.setItem('tower_last_user_id', session.user.id)
  setSession(session)
```

The existing `signOut()` keeps `unsafeResetDatabase()` as a best-effort cleanup (unchanged). The on-sign-in check is the safety net.

**Files:** `src/auth/AuthContext.tsx`
**Dependencies:** `@react-native-async-storage/async-storage` (already in project via WatermelonDB)


## 5. Financial Wellness Score

### Concept
The user allocates their income into named buckets (e.g., Food = 15%, Transport = 10%). The score measures how closely actual spending tracks those targets. Score is 0–100. A score of 74 means the user is on plan for 74% of their allocated budget.

### Score Algorithm

```
for each budget category with target_pct set:
  target_spend = monthly_income × (target_pct / 100)
  actual_spend = sum of transactions in this category this month
  ratio = actual_spend / target_spend          // 0 = nothing spent, 1 = on budget, >1 = over
  category_score = clamp(1 - max(0, ratio - 1), 0, 1)
    // = 1.0 if at or under budget
    // = 0.5 if 50% over budget
    // = 0.0 if 100% over budget

global_score = (Σ category_score_i × target_pct_i) / (Σ target_pct_i) × 100
```

- Categories with no `target_pct` are excluded from the score
- Score is a `useMemo` — recomputed live from transaction data, never stored
- `monthly_income` = `useMonthlyIncome(transactions)` (existing hook)

### Sparkline History
- The score is recomputed for each of the last 7 days (or 30 days in month mode) using only transactions up to that day
- `useWellnessScore` returns `{ score: number, history: number[], delta: number }`
  - `history`: array of daily scores ending today (7 or 30 values)
  - `delta`: today's score minus 7 days ago (positive = improving)

### Status Labels
| Score | Label | Color |
|---|---|---|
| 85–100 | Excellent | #4ade80 |
| 70–84 | Good | #f59e0b |
| 50–69 | Fair | #fb923c |
| 0–49 | At risk | #ef4444 |

### Hook Interface
```typescript
interface WellnessResult {
  score: number;          // 0–100, integer
  history: number[];      // daily scores, oldest first, length 7 or 30
  delta: number;          // score change vs 7 days ago
  status: string;         // 'Excellent' | 'Good' | 'Fair' | 'At risk'
  statusColor: string;
}

function useWellnessScore(
  transactions: Transaction[],
  budgets: BudgetCategory[],
  monthlyIncome: number
): WellnessResult
```

**File:** `src/hooks/useWellnessScore.ts`

## 6. Plan Tab Restructure

### Layout
The Plan tab has two views toggled by a segmented control at the top: **Report** and **Planning**.

### Report View
- Section label: `WHERE IT WENT · [Month Year]`
- Compact summary bars: one row per budget category (name, amount, colored progress bar). Over-budget rows show ⚠ in red.
- "Open full analysis →" row at the bottom: taps into the existing `SankeyChart` inside a modal or a full-screen push (`pageSheet` presentation)

### Planning View

**Section 1 — Income Buckets** (expandable row)
- Row header: "💰 Income Buckets" + count of buckets + chevron
- Tap to expand inline (accordion) or open as a bottom sheet
- Expanded content:
  - One row per budget category: emoji + name on left, editable % field on right (tapping opens number pad)
  - Running total: "Allocated: 30% of income" (updates live as user edits)
  - "＋ Add bucket" button: opens a small form (name, emoji, color picker, % amount, monthly dollar limit)
  - Long-press a row → delete confirmation
  - Save button: calls `updateBudget(id, { targetPct, monthlyLimit })` for each changed row
- Saving a bucket's `target_pct` immediately updates the wellness score

**Section 2 — Savings Goals** (expandable row)
- Row header: "🎯 Savings Goals" + count + chevron
- Expand inline or bottom sheet (same pattern as Buckets)
- Expanded content: existing goal cards (name, emoji, progress bar, projected date)
- "＋ Add goal" at bottom (existing modal, unchanged)
- Long-press → delete

### Supabase Schema Change
```sql
ALTER TABLE budget_categories ADD COLUMN target_pct FLOAT;
-- nullable; null means this category is excluded from wellness score
```

No WatermelonDB schema change needed — `target_pct` lives only in Supabase (it's plan metadata, not financial data).

**Updated `createBudget` / `updateBudget`:**
```typescript
export async function updateBudget(
  id: string,
  fields: { monthlyLimit?: number; targetPct?: number }
): Promise<void>
```

## 7. Modern Floating Nav Bar

Replace the standard `TabBar` with a custom floating pill component.

### Spec
- Container: `border-radius: 24px`, `backgroundColor: #1e293b`, `borderWidth: 1`, `borderColor: #334155`
- Shadow: `elevation: 16` / `shadowColor: #000`, `shadowOpacity: 0.4`, `shadowRadius: 16`
- Positioned: `position: absolute`, `bottom: 20px` (above safe area), `left: 20px`, `right: 20px`
- Each tab: icon (emoji or SVG icon, 20px) + label (8px, bold when active)
- Active tab label: `#6366f1`; inactive: `#475569`
- Active icon gets a small indigo underline dot (4px circle)
- Tabs: Home (house icon), Plan (chart icon), Settings (gear icon)

**Implementation:** Custom `tabBar` prop on `<Tab.Navigator>` rendering a `TouchableOpacity` per tab. Uses `useSafeAreaInsets` for bottom padding.

**File:** `src/navigation/FloatingTabBar.tsx` (new)
